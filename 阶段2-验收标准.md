# Cognitive Runtime Foundation

## 第二阶段验收标准（v0.1）

第二阶段的验收目标不是：

```text id="5y6mwx"
AI 更聪明
```

而是：

# “认知 Runtime 是否真正成立”

即：

```text id="a0xjol"
世界信息
→ 主观感知
→ 注意力筛选
→ 记忆
→ 规律发现
→ 知识形成
```

是否形成：

# 可扩展认知闭环。

---

# 一、总体验收原则

第二阶段通过的核心标准：

```text id="ep0dgj"
新增认知现象
不需要修改底层系统
只需要新增：
Tag
Rule
Hypothesis
Knowledge Edge
```

如果后面：

```text id="d7g24l"
新增恐惧
新增好奇
新增火认知
新增动物认知
```

还需要：

```text id="aqo6i7"
改 PerceptionSystem
改 MemorySystem
改 KnowledgeSystem
```

则：

# 第二阶段失败。

---

# 二、P0 验收（必须通过）

---

# 1. Subjective Perception（主观感知）

## 验收要求

Agent：

```text id="4vggq0"
不能直接读取 WorldState
```

必须经过：

```text id="b3ij8k"
PerceivedStimulus
```

---

## 验收场景

### 场景 A

```text id="lh9ucd"
世界中存在 Smoke field
```

但：

```text id="hnzy0e"
Agent 距离太远
```

结果：

```text id="n3s8m9"
无 SmokeStimulus
```

---

### 场景 B

两个 Agent：

```text id="p9b0iw"
一个靠近火
一个远离火
```

结果：

```text id="0wn6ij"
感知结果不同
```

---

## 判定

如果：

```text id="hmyiuk"
所有 Agent 看到相同世界
```

则：

# 不通过。

---

# 2. Attention Filtering（注意力筛选）

## 验收要求

同一时刻多个刺激：

```text id="t4vzlk"
不能全部进入记忆
```

必须竞争。

---

## 验收场景

Agent：

```text id="g80x0u"
饥饿 + 附近火 + 远处野兽
```

---

结果：

```text id="jzd6ec"
不同 Bias 下 FocusedStimulus 不同
```

例如：

```text id="zdbup4"
高 HungerBias → 食物优先
高 FearBias → 野兽优先
```

---

## 判定

如果：

```text id="g1v8fp"
所有刺激全部进入记忆
```

则：

# 不通过。

---

# 3. Memory Formation（记忆形成）

## 验收要求

记忆：

```text id="6q0tzg"
必须来自感知结果
```

不是直接写入。

---

## 验收场景

```text id="s99v3r"
闻到烟
→ 形成 SmelledSmoke Memory
```

---

```text id="85m9is"
接近火并受伤
→ 形成 TookDamageFromFire Memory
```

---

## 判定

如果：

```text id="zeb4ja"
Memory 直接读取 WorldState
```

则：

# 不通过。

---

# 4. Forgetting（遗忘）

## 验收要求

记忆：

```text id="vj0gxh"
必须衰减
```

否则：

# 系统最终爆炸。

---

## 验收场景

```text id="o6j3bh"
普通事件长期未强化
→ 消失
```

---

```text id="zdlm2x"
高 emotionalWeight
→ 保留更久
```

---

## 判定

如果：

```text id="j26dbk"
Memory 无限增长
```

则：

# 不通过。

---

# 5. Hypothesis Generation（规律发现）

这是第二阶段最重要验收。

---

## 验收要求

系统：

```text id="xh8hwj"
必须能从重复事件中形成假设
```

不是手动解锁。

---

## 验收场景

多次：

```text id="k3rnp4"
Smoke
→ 附近有 Fire
```

结果：

```text id="pw8n35"
生成：
SmokeMeansFireNearby
```

---

## 判定

如果：

```text id="8wq95c"
Knowledge 是手动 hardcode
```

则：

# 不通过。

---

# 6. Knowledge Graph（知识图谱）

## 验收要求

知识：

```text id="s2h5fh"
必须是图结构
```

不是：

```text id="trz0wp"
bool unlocked
```

---

## 验收场景

```text id="zlhrim"
Fire
 ├─ Warmth
 ├─ Danger
 ├─ Protection
```

不同 Agent：

```text id="h2b7nz"
图谱不同
```

---

## 判定

如果：

```text id="r6mx8p"
Knowledge = bool
```

则：

# 不通过。

---

# 三、P1 验收（强烈建议）

---

# 7. Wrong Knowledge（错误认知）

## 验收要求

知识：

```text id="vjlwmu"
允许错误
```

---

## 验收场景

某 Agent：

```text id="rb0znq"
连续：
雷声 → 死亡
```

形成：

```text id="7r1vb8"
Thunder = Danger
```

即使：

```text id="u9n4rb"
逻辑不完全正确
```

---

# 判定

如果：

```text id="jlwm3w"
所有知识永远正确
```

则：

# 不是真正认知系统。

---

# 8. Knowledge Reinforcement（知识强化）

## 验收要求

重复成功：

```text id="1gz6ih"
confidence 上升
```

长期失败：

```text id="29a8xv"
confidence 下降
```

---

# 四、P2 验收（第三阶段前）

---

# 9. Individual Difference（认知差异）

不同 Agent：

```text id="qyk7n0"
知识图谱不同
```

---

# 10. Social Propagation（传播）

知识：

```text id="ep5dxf"
可传播
```

但：

```text id="5my48u"
不要求现在完成
```

---

# 五、真正核心验收

第二阶段真正成功的标准：

不是：

```text id="z6st8p"
角色会不会用火
```

而是：

# “系统是否允许火认知自然形成”

---

# 六、最终一句话验收标准

```text id="t9pb1j"
第二阶段通过的标准：

世界中的重复经历，
能够被个体主观感知、
记忆、
关联、
并沉淀为可变化的知识。
```
