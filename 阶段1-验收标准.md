# 怎么测试验证

## 1. 编译验证

研发需要保证：

```bash
cmake -S . -B build
cmake --build build
```

能成功。

---

## 2. 启动验证

运行：

```bash
./human_evolution_headless
```

应能看到 Tick 持续推进。

验收标准：

```text
程序不崩溃
Tick 正常递增
WorldState 正常变化
```

---

## 3. 确定性验证

同一个 Seed，跑 1000 Tick，两次结果必须完全一致。

示例：

```bash
./human_evolution_headless --seed 12345 --ticks 1000 --dump run_a.json
./human_evolution_headless --seed 12345 --ticks 1000 --dump run_b.json
```

验收：

```text
run_a.json == run_b.json
```

这是最重要的测试。

---

## 4. 非确定性验证

不同 Seed，结果应该不同。

```bash
./human_evolution_headless --seed 111 --ticks 1000 --dump a.json
./human_evolution_headless --seed 222 --ticks 1000 --dump b.json
```

验收：

```text
a.json != b.json
```

---

## 5. Grid 测试

验证：

```text
坐标访问正确
边界判断正确
越界不会静默污染内存
```

例子：

```text
Grid(10,10)
At(0,0) 正常
At(9,9) 正常
InBounds(-1,0) false
InBounds(10,0) false
```

---

## 6. Random 测试

验证：

```text
同一个 seed 输出序列一致
不同 seed 输出序列不同
Next01 永远在 0~1
```

---

## 7. Tick 测试

验证：

```text
初始 Tick = 0
执行一次 Scheduler.Tick 后 Tick = 1
执行 100 次后 Tick = 100
```

---

## 8. System 顺序测试

系统执行顺序必须固定。

测试方法：

```text
创建 TestSystemA / B / C
每个系统写入日志
执行一次 Tick
日志必须是 A → B → C
```

---

## 9. 气味系统测试

创建固定场景：

```text
地图 10x10
气味源在 5,5
风向向东
跑 10 Tick
```

验收：

```text
东侧气味强于西侧
气味总量会衰减
Agent 靠近后能检测到气味
```

---

## 10. 火系统测试

创建固定场景：

```text
地图 10x10
5,5 有火
周围格子可燃
```

验收：

```text
火会提升周边温度
火有概率扩散
湿度高时扩散更慢
燃尽后火强度下降
```

---

## 11. Agent 测试

创建固定场景：

```text
Agent 在 1,1
食物气味源在 8,1
无危险
```

验收：

```text
Agent 能感知气味
Agent 会向气味强度更高方向移动
Agent 饥饿值影响决策
```

---

# 最关键验收标准

第一版必须证明三件事：

```text
1. 世界能稳定 Tick
2. 同 Seed 可重放
3. 系统可以继续扩展而不重构
```

这三件事成立，地基就没有塌。
