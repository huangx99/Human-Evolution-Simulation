# Human Evolution Simulation Core

## 底层架构设计文档（V0.1）

---

# 1. 项目目标

本项目不是传统脚本驱动游戏。

目标是构建：

```text
一个可长期扩展的文明演化模拟核心
```

核心理念：

```text
环境变化
→ 信息传播
→ 生物感知
→ 行为决策
→ 改变环境
→ 形成记忆
→ 群体传播
→ 文明沉淀
```

项目必须支持：

* 长期文明演化
* 大规模系统扩展
* 回放
* AI训练
* 多前端接入
* Dedicated Server
* Headless Simulation
* Deterministic Simulation

因此：

```text
模拟层必须独立于渲染层
```

---

# 2. 总体架构

```text
Frontend（任意前端）
    ↓
API Layer
    ↓
Simulation Runtime
    ↓
Simulation Systems
    ↓
World State
```

---

# 3. 架构铁律（必须遵守）

## 3.1 模拟层不能依赖任何引擎

禁止：

```text
Unity API
UE API
Renderer
Audio
Input
Animation
```

Simulation Core 必须纯 C++。

---

## 3.2 所有系统必须确定性

禁止：

```cpp
rand()
std::random_device
系统时间
线程竞争导致随机结果
```

所有随机必须来自统一 Seeded Random。

---

## 3.3 世界只能通过 Tick 推进

世界时间：

```text
Tick 0
Tick 1
Tick 2
```

禁止依赖真实时间。

---

## 3.4 系统之间禁止直接调用

错误：

```cpp
FireSystem -> SmellSystem
```

正确：

```text
所有系统通过 WorldState 读写世界
```

---

## 3.5 数据与规则分离

禁止硬编码：

```cpp
if (forest)
```

应使用：

```json
BiomeRules
ReactionRules
SpeciesRules
```

---

## 3.6 表现层只能观察世界

前端：

```text
读取 Snapshot
发送 Command
```

禁止直接修改 WorldState。

# 7. 时间系统

文件：

```text
core/time/sim_clock.h
```

结构：

```cpp
struct SimClock
{
    Tick currentTick = 0;
    f32 tickDeltaSeconds = 1.0f;

    void Step()
    {
        currentTick++;
    }
};
```

规则：

* 世界只能通过 Tick 推进
* 所有系统基于 Tick 运算
* 禁止依赖真实时间

# 10. Grid 系统

文件：

```text
core/container/grid.h
```

结构：

```cpp
template<typename T>
class Grid
{
public:
    Grid(i32 width, i32 height);

    bool InBounds(i32 x, i32 y) const;

    T& At(i32 x, i32 y);

    const T& At(i32 x, i32 y) const;

private:
    i32 width;
    i32 height;
    std::vector<T> data;
};
```

Grid 是整个世界模拟基础。

---

# 11. WorldState（核心世界状态）

文件：

```text
sim/world/world_state.h
```

结构：

```cpp
struct WorldState
{
    SimClock clock;
    Random random;

    i32 width;
    i32 height;

    Grid<f32> temperature;
    Grid<f32> humidity;
    Grid<f32> fire;
    Grid<f32> smell;
};
```

规则：

* 所有系统围绕 WorldState 工作
* 系统不得保存世界副本
* 世界数据集中存储

# 13. Scheduler

文件：

```text
sim/scheduler/scheduler.h
```

结构：

```cpp
class Scheduler
{
public:
    void AddSystem(std::unique_ptr<ISystem> system);

    void Tick(WorldState& world);

private:
    std::vector<std::unique_ptr<ISystem>> systems;
};
```

规则：

* 系统执行顺序统一由 Scheduler 管理
* 系统之间不得互调

# 16. 信息场（核心设计）

项目核心不是角色。

核心是：

```text
信息传播
```

因此世界由多个 Field 组成。

例如：

```text
TemperatureField
HumidityField
SmellField
DangerField
SoundField
```

Agent 不直接“看世界”。

Agent 读取：

```text
Field 信息
```
